CORSを消すのではなく、プロキシを試すUI

静的HTMLでも、ちゃんと“プロダクト感”は出せる。

この画面はブラウザから直接CORSを魔法で突破するものではありません。 その代わり、プロキシ経由の送信を見た目よく試せるようにした、かなりそれっぽい検証UIです。 便利な顔をした道具は、だいたい裏で誰かが苦労しています。

リクエスト作成

GET / POST / PUT / PATCH / DELETE
想定しているプロキシ形式は、/{target-url} に中継するタイプです。 例えば http://localhost:8080/https://api.example.com/data のような形です。 バックエンドの仕様に合わせて調整してください。
